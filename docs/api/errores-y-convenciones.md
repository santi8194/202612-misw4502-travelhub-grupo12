# Errores y convenciones de API

## Convenciones mínimas

- Usar códigos HTTP estándar.
- Exponer endpoints de salud para servicios desplegables.
- Mantener rutas y payloads documentados localmente antes de incorporarlos al catálogo global.
- Evitar definir contratos distintos para el mismo concepto entre servicios.

## Estado actual

El repositorio todavía no presenta un contrato transversal único de error. Hasta que exista uno, cada servicio debe documentar las respuestas relevantes en su README o colección Postman.
